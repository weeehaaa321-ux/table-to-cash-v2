2
Receipt printer — Xprinter XB-80T as a Windows service
The most important step. When this is right, the cafe runs without you.

Goal: a Windows service called TTC-PrintAgent running on the cashier PC. It bridges the cashier's browser → printer over the LAN, auto-starts on boot, auto-restarts on crash. No PowerShell window left open. Survives reboots forever.

All commands in this step run on the cashier PC, in PowerShell as Administrator. Open it once: Start → type powershell → right-click → "Run as administrator". Keep it open through every sub-step (or reopen as Admin after Chocolatey, where noted).
Get the printer's IP printer

Power off the Xprinter XB-80T (switch on the side). Hold the small FEED button on top. Power back on while still holding FEED. After ~3 seconds, release. A self-test page prints.

Find the line IP Address: x.x.x.x. Type it into the Printer IP field at the top of this page — every later command auto-fills with it.

This is non-destructive — the existing print system reconnects automatically. Nothing breaks.
Verify the cashier PC can reach the printer cashier PC 
ping {{IP}}

"Reply from..." → ✅. "Request timed out" → cashier PC and printer aren't on the same LAN. Check both Ethernet cables go to the same router.

Install Node.js LTS cashier PC

First check if it's already installed:

node --version

If it prints v18.x, v20.x, or v22.x → already installed, skip the rest. Otherwise:

  1. Open browser on cashier PC → https://nodejs.org
  2. Click the green LTS button → run the downloaded .msi
  3. Click Next through every screen → Install → Finish
  4. Close PowerShell, open a fresh Admin PowerShell (PATH only updates in new shells)
  5. Run node --version again — should now print a version
Install Chocolatey cashier PC

Skip if choco --version already prints a number.

Set-ExecutionPolicy Bypass -Scope Process -Force; iwr https://chocolatey.org/install.ps1 -UseBasicParsing | iex
After it finishes, close PowerShell and reopen as Admin — Chocolatey added a PATH variable that needs a fresh shell.
Install NSSM (the service wrapper) cashier PC 
choco install nssm -y
Download print-agent.mjs to Desktop cashier PC 
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/weeehaaa321-ux/table-to-cash-v2/master/scripts/print-agent.mjs" -OutFile "$env:USERPROFILE\Desktop\print-agent.mjs"

Verify it landed:

Test-Path "$env:USERPROFILE\Desktop\print-agent.mjs"

Should print True.

Install the service cashier PC 
nssm install TTC-PrintAgent "C:\Program Files\nodejs\node.exe" "$env:USERPROFILE\Desktop\print-agent.mjs"
Set printer IP and Vercel URL on the service cashier PC 
nssm set TTC-PrintAgent AppEnvironmentExtra "PRINTER_IP={{IP}}" "APP_URL={{URL}}"
⚠ Fill in Printer IP and v2 Vercel URL at the top of this page first — the command above is using placeholders.
Start the service cashier PC 
nssm start TTC-PrintAgent

Confirm it's running:

Get-Service TTC-PrintAgent

Should show Running. If Stopped, env vars are wrong — rerun sub-step 2-8 with the correct values, then start again.

Reboot test — the truth test cashier PC

Reboot the cashier PC. After it boots back up, open the cashier UI → click 🖨 on any bill → receipt prints.

If yes, the service is properly auto-starting. This is what lets you walk away.

If it fails after reboot: Get-Service TTC-PrintAgent in Admin PowerShell. If "Stopped", run nssm start TTC-PrintAgent. If still failing, run nssm edit TTC-PrintAgent and verify the path to node.exe and print-agent.mjs are correct.
